    Simplicity

    Using properties instead of classes

    V1 Buttons
    While this is traditional what every framework does, it can become quite cluttered the more classes you add

    Classes can get
    complex and cluttered

    Move things to properties instead

    Easier to understand and less markup

    Easy to overide with classes

    Properties provide a simple and easy way to do the same thing as classes, but make your markup easier to understand

    
      <button block>Default</button>
      <button secondary block>Secondary</button>
      <button danger block>Danger</button>
      <button light block>Light</button>
      <button dark block>Dark</button>

    Equally easy to style and extend

    
  [round]{
    border-radius: 30px;
  }
  [button][block]{
    width: 80%;
  }

    Ionic 2 uses element properties

    Less clutter, easier to reason, and just as customizable

    Doesn't change how you write your styles

    Ionic is using element properties to keep your markup clean and simple, but you can still easily use CSS classes just like you always did.

    V1's Routing

    
.config(function($stateProvider){
  $stateProvider
    .state('firstpage', {
      url:'/firstpage',
      templateUrl: 'firstpage.html'
    })
    .state('secondpage', {
      url:'/secondpage',
      templateUrl: 'secondpage.html'
    })
    .state('thirdpage', {
      url:'/thirdpage/:id',
      templateUrl: 'thirdpage.html'
    });
});

    UI-Router

    Doesn't give the flexibility you want

    Navigation is more about a user experience

    Traditional browser-based URLs don't offer that

    Developers NEED to be in complete control

    Push/Pop Navigation

    
  import SecondPage from '../secondpage';
  export class FirstPage {
    constructor(nav: NavController) {
      this.nav = nav
    };
    pushState() {
      this.nav.push(SecondPage)
    };
  };
  • Heavily inspired by native implimentations
  • Much more flexible an mature navigation pattern

    Building Ionic with new standards

    JavaScript has changed

    Ionic 1 was built with ES5

    ES6 brings many new features

    NG2 is built on top of ES6

    Different look, but all JS Classes

  • ES5 has served us well
  • ES6 gives us many new features

    • Like javascript has grown up

    Benefit of ES6

    Modular: Import what you need & and gets rid of globals

    Scoping: let defines a block-scoped variable

    Promises: Built into the language natively

    Classes: Simpler syntax for building objects

    Modules: proper module loading prevents polution the global scope

    Also able to generate a build of code that is speific to what you import

    All these features that were in ng1, now are baked into js
